SQL基础整理

@wentixiaogege · Jun 23, 2019 · 6 min read

SQL基础知识整理。

一、什么是SQL

1.数据库（database)

保存有组织的数据的容器（通常是一个文件或者一组文件）

2.数据库软件（DBMS，数据库管理系统）

用来创建和操控数据库

3.表（table）

某种特定类型数据的结构化清单，同一个数据库中表名唯一

4.模式（schema）

关于数据库和表的布局及特性的信息

5.列（column）

表中的一个字段，所有表都是由一个或者多个列组成（将数据库想象成一个表格，网格中的每一列存储着一条特定的信息）

6.数据类型（datatype）

所容许的数据类型。每个表列都有相对应的数据类型，它限制或容许该列中存储的数

7.行（row）

表中的一个记录

8.主键（primary key）

一列（或者一组列），其值能够唯一区分开每个行

9.SQL（structured query language）

结构化查询语言，专门用来与数据库通信

二、MYSQL简介及一些基础语句

1.MySQL

MySQL是一种DBMS，database management systerm，数据库软件，基于客户机-服务器运行

2.Navicat For MySQL

一款简单好用的数据库可视化软件（方便，点击连接，输入主机名、端口、用户名、密码等即可，点击表及可视） 也可以用Sequel Pro

3.关键字（key word）

作为Mysql语言组成部分的一个保留字，不允许用关键字命名一个表或一个列。

4.选择数据库

USE crashcource；
# 选择使用crashcource数据库

5.返回可用数据库的一个列表

SHOW DATABASES；

6.获取一个数据库内的表的列表

SHOW  TABLES；

7.获取表列名

SHOW COLUMNS FROM CUSTOMERS；
# 返回字段名、数据类型、是否允许NULL、键信息、默认值以及其他信息

8.自动增量

MySQL可以自动为下一个行分配下一个可用编号，不用在添加一行时手动分配唯一值。启用需在用CREATE语句创建表时将它作为表定义的组成部分

9.DESCRIBE语句

DESCRIBE作为SHOW COLUMNS FROM的一种快捷方式。

比如DESCRIBE customers；是 SHOW COLUMNS FROM customers；的一种快捷方式

10.支持的其他SHOW语句

SHOW STATUS；# 显示广泛的服务器状态信息
SHOW CREATE DATABASE；
SHOW CREATE TABLES；
# 分别用来显示创建特定数据库或者表的MySql语句
SHOW GRANTS；
# 显示授予用户（所有用户或特定用户）的安全权限
SHOW ERRORS；
SHOW WARNINGS；
# 用来显示服务器错误或者警告

三、检索数据

1.检索单个列

SELECT prod_name
FROM products;
# 从products表中检索一个名为prod_name的列，所需列名在SELECT关键字之后给出

2.在mysql命令行中必须采用；结束语句

3.SQL不区分大小写，但是许多SQL开发人员喜欢对SQL关键字使用大写，而对所有列、表名使用小写，这样方便代码阅读和调试，同样的，适当使用空格分行也为了这个目的

4.检索多个列

SELECT prod_id,prod_name,prod_price
FROM products;

5.检索所有列

SELECT *
FROM products；
# 注意，检索不必要的额列通常会降低检索和应用程序的性能

6.检索不同的行

SELECT DISTINCT vend_id
FRPM products;
# 返回批次不同的行

7.限制查询结果的数量或位置

SELECT prod_name
FROM products
LIMIT 5;
# 可用来测试
SELECT prod_name
FROM products
LIMIT 5，5;
# 指示MySQL返回从行5开始的5行，第一个数为开始的位置，第二个位置为要检索的行数
# LIMIT 4  OFFSET 3与LIMIT 3，4一样
注意：检索出来的第一行为行0而不是行1，LIMIT 1，1；检索出来的是第二行

8.完全限定表名

SELECT products.prod_name
FROM crashcource.products;
#熟悉一下这个语法

四、排序检索数据

1.子句（clause）

SQL由子句构成，有些子句是必需的，有些是可选的。一个子句通常由关键字和所提供的数据组成。子句例子有SELECT语句FROM子句

2.ORDER BY子句字母顺序排序

SELECT prod_name
FROM products
ORDER BY prod_name;

3.按多个列排序

SELECT prod_id,prod_price,prod_name
FROM products
ORDER BY prod_price,prod_name;
#逐层下沉

4.指定方向排序

SELECT prod_id,prod_price,prod_name
FROM products
ORDER BY prod_price DESC,prod_name;
#价格降序排序商品
#与DESC相反的关键词是ASC（ASCENDING0），升序排序，但因为升序是默认的用户不大

5.利用刚刚学习的寻找一列中最大的值

SELECT prod_price
FROM products
ORDER BY prod_price DESC
LIMIT 1;

五、过滤数据

1.搜索条件（search criteria），也成为过滤条件（filter condition）

2.条件过滤

SELECT prod_price
FROM products
WHERE prod_price>=10;

3.WHERE子句的位置

在同时使用ORDER BY和WHERE子句时，应该让ORDER BY位于WHERE之后，否则会产生错误

4.常见的WHERE子句操作符

= 等于    
<> 不等于   
!= 不等于  
=< 小（等）于  
>= 大（等）于  
BETWEEN 两者之间
注意：在执行匹配操作时候不区分大小写

SELECT prod_price
FROM products
WHERE prod_name='chengzi';
注意：单引号用来限制字符串，如果将值与串类型的列进行比较，则需要限定引号

5.区间查询

SELECT prod_name,prod_price
FROM products
WHERE prod_price BETWEEN 5 AND 10;

6.NULL值（no value）

不包含值，它与字段包含0、空字符串或仅仅包含空格不同

SELECT prod_name,prod_price
FROM products
WHERE prod_price IS NULL;
# NULL与不匹配是两种含义

六、数据过滤

1.操作符（operator）

用来联结或改变WHERE子句中的子句的关键字，也成为逻辑操作符（logical operetor）

2.AND操作符

SELECT prod_id,prod_price,prod_name
FROM products
WHERE ven_id=1003 and  prod_price<=10;

3.OR操作符

SELECT prod_id,prod_price,prod_name
FROM products
WHERE ven_id=1003 or ven_id=1004;

4.AND在计算次序中优先级较高，为避免操作符被错位组合，可以写如下语句

SELECT prod_id,prod_price,prod_name
FROM products
WHERE （ven_id=1003 or ven_id=1004） and prod_price>=10;
#圆括号比AND和OR优先级更高

5.IN操作符

SELECT prod_id,prod_price,prod_name
FROM products
WHERE ven_id IN (1002,1003)
ORDER BY prod_name;

6.NOT操作符

SELECT prod_id,prod_price,prod_name
FROM products
WHERE ven_id IN (1002,1003)
ORDER BY prod_name;
注意：MySQL支持NOT对IN、BETWEEN和EXISTS子句相反，这与其他DBMS允许使用NOT对各种条件取反有很大的差别

七、通配符过滤

1.通配符（wildcard）

用来匹配值的一部分的特殊字符

2.搜索模式（search pattern）

由字面值、通配符或两者组合构成的搜索条件

3.特定查询%通配符

SELECT prod_id,prod_name
FROM products
WHERE prod_name LIKE 'jet%';
# 检索任意jet开头的词，且搜索模式区分大小写，与Jet并不匹配
SELECT prod_id,prod_name
FROM products
WHERE prod_name LIKE '%anvil%';
# 包括anvil的所有值
SELECT prod_id,prod_name
FROM products
WHERE prod_name LIKE 'a%s';
# %代表搜索模式中给定位置的0个、1个或多个字符

注意： 1.尾空格可能会干扰通配符匹配 2.通配符不支持匹配NULL

4._通配符

_只能匹配单个字符而不是多个字符,不能多也不能少

SELECT prod_id,prod_name
FROM products
WHERE prod_name LIKE '_jet';

5.通配符使用技巧

通配符搜索的处理一般比其他的搜索时间更长。不要过度使用通配符,将通配符用在搜索模式的开始处是最慢的，仔细注意通配符的位置

八、正则表达式搜索

1.正则表达式

用来匹配文本的特殊的串；所有种类的程序设计语言、文本编辑器、操作系统等都支持正则表达式。

2.基本字符匹配

SELECT prod_name
FROM products
WHERE prod_name REGEXP'1000'
ORDER BY prod_name;
# REGEXP后面跟的就是正则表达式（与正文1000匹配的一个正则表达式)处理

SELECT prod_name
FROM products
WHERE prod_name REGEXP'.000'
ORDER BY prod_name;
# 其中.表示匹配任意一个字符

3.使用BINMARY区分大小写

where prod_name REGEXP BINMAY 'JetPack  .000'
# MySQL本身不区分大小写

4.进行OR匹配

SELECT prod_name
FROM products
WHERE prod_name REGEXP '1000|2000'
ORDER BY prod_name;

5.匹配几个字符之一

SELECT prod_name
FROM products
WHERE prod_name REGEXP '[123] Ton'
ORDER BY prod_name;
# 意思是匹配1或者2或者3，[]是另一种形式的OR语句，与[1|2|3]一致
# [^123]匹配除这些字符以外的东西

6.匹配范围

SELECT prod_name
FROM products
WHERE prod_name REGEXP '[1-5] Ton'
ORDER BY prod_name;

7.匹配特殊字符

SELECT prod_name
FROM products
WHERE prod_name REGEXP '\\.'
ORDER BY prod_name;
# \\为前导，转义符

# 转义符的其他用处：
# \\f  换页   \\n  换行  \\r  换车   
# \\t  制表   \\v   纵向制表   \\\  转义\

8.匹配字符类（character class）

  • [:alnum:] 任意字母或数字
  • [:alpha:] 任意字符
  • [:blank:] 空格和制表
  • [:cntrl:] ASCⅡ控制字符
  • [:digit:] 任意数字
  • [:graph:] 与[:print:]相同，但不包括空格
  • [:lower:] 任意小写字母
  • [:print:] 任意可打印字符
  • [:punct:] 不在[:cntrl:] 和[:alnum:]中的字符
  • [:space:] 包括空格字符在的任意空白字符
  • [:upper:] 任意大写字母
  • [:xdigit:] 任意16进制数字

9.匹配多个实例

  • \* ：0个或者多个匹配
  • \+ ：1个或者多个匹配{1，}
  • ? ：0个或者多个匹配{0，1}
  • {n} ：指定数目的匹配
  • {n,} ：不少于指定数目的匹配

  • {n,m} ：匹配数据的范围

    # 举例
SELECT  prod_name
FROM products
WHERE prod_name REGEXP '\\([0-9] sticks?\\)'
ORDER BY  prod_name;
# \\(匹配），[0-9]匹配任何数字，sticks？匹配stick和sticks（？使s出现0次或者1次），\\（匹配）
SELECT  prod_name
FROM products
WHERE prod_name REGEXP '[[:digit:]]{4}'
ORDER BY prod_name;
# 连在一起的任意4位数字

10.定位符

定位元字符：

  • ^ 文本的开始
  • $ 文本的结尾
  • [[:<:]] 词的开始

  • [[:>:]] 词的结尾

    SELECT  prod_name
FROM products
WHERE prod_name REGEXP '^[0-9]\\.)'
ORDER BY  prod_name;

九、创建计算字段（转换、计算、格式化）

1.字段（field）

基本上与列（column）的意思相同，经常互换使用，术语字段通常用在计算字段的连接上。

2.Concat()（ 连接函数）

SELECT Concat(vend_name,'  (',vend_country, ')')
FROM vendors
ORDER BY vend_name;

3.RTrim() （删除右侧多余的空格）

SELECT Concat(RTrim(vend_name),'  (',RTrim(vend_country), ')')
FROM vendors
ORDER BY vend_name;
#LTrim()去掉左边多余的空格

4.AS（使用别名）

SELECT Concat(RTrim(vend_name),'  (',RTrim(vend_country), ')')
AS vend_title
FROM vendors
ORDER BY vend_name;

别名的其他用途：别名还有其他用途。常见的用途是在实际的表列名包含不符合规定的字符（如空格）时重新命名他，在原来的名字含混淆或容易误解时扩充它等等。

有时别名也被称为导出列（derived column）

5.算术运算

+ - * / 加减乘除

6.测试运算

省略FROM语句可以简单地访问和处理表达式，如SELECT 3*2；将返回6，SELECT Trim（’abc ‘）;将返回abc，SELECT Now();返回当前地日期和时间

十、使用函数处理数据

1.文本处理函数

SELECT   Upper(vend_name) AS vend_name_upcase
FROM vendors
ORDER BY vend_name;

Upper()转换大写函数
Left() 返回串左边地字符

Locate() 找出串的一个子串

LTrim() 去掉串左边的空格
Right() 返回串右边的字符
RTrim() 去掉串右边的空格

SOUNDEX是一个将任何文本串转换成其语音表示的字母数字模式的算法，姓名模糊查询）

Month() 返回一个时间的月份部分
Now() 返回当前时间
Second() 返回一个时间的秒部分
Time() 返回一个日期时间的时间部分
Year() 返回一个日期的年份部分

3.数值处理函数

Abs() 返回一个数的绝对值
Cos() 返回一个角度的余弦值
Exp() 返回一个数的指数值

Pi() 返回圆周率
